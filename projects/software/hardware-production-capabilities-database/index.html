Many volunteers are offering 3D printing, CNC and other production services but currently we have no overview. When designs need te be rolled out we need people know what local production capability there is and how to contact them.

üë®‚Äçüî¨Hypothesis

ü§î Assumptions

Assumption 1, assuming we need a more structured way of cataloging production volunteers.

Assumption 2, I think this problem is not yet tackled by other proposals. I'm sensing some overlap with webappresource_management and the several overarching project management proposals.

üíªImplementation Suggestion

V1

What it is

Basic database (online spreadsheet) where individuals can volunteer their production capabilities.

Registering:

  • Type (CNC, 3D printing, other)
  • Location
  • Contact information (slack handle?)

What it is not

It is not a go all place to offer volunteer help. It would just shown volunteers able to help with producing proven design and prototypes.

V1 will be about storing data, no visualization yet.

Design

Online spreadsheet to get the idea: https://docs.google.com/spreadsheets/d/1DIdCv0yNLarHZKBnzuomJb5OAwem8KpMSLcDqbwfvYo/edit?usp=sharing

Data/Experiments

Need access to form submission about production capabilities.

V2

Visualization of available production capabilities. Probably an interactive map.

  • Reduce slack and other channel clutter.
